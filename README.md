## 株式会社SmartHR（2023/06〜現在）

### 年末調整プロダクト開発（2023/06〜現在）
年末調整プロダクトの開発に携わり、バックエンド、フロントエンド、インフラの幅広い領域で貢献しています。

#### 環境
- 開発手法
  - スクラム
- チーム構成
  - PM：1人
  - ドメインエキスパート：1人
  - プロダクトエンジニア：6人
  - QAエンジニア：1人
  - UXライター：1人
- 役割: プロダクトエンジニア
- 使用技術: Rails, React, Ruby, TypeScript, Google Cloud(Cloud Run, Cloud Storage, Cloud SQL, App Engine), Terraform, PostgreSQL, CircleCI, GitHub Actions, New Relic, Sentry, GitHub, Jira, Slack

#### 主な業務と成果
- バックエンド開発（Rails）
  - API設計と実装
  - パフォーマンス改善
    - 同期処理
      - 12倍の改善(3分4秒→15秒)
    - 非同期処理
      - 2.4倍の改善(52分28秒→22分1秒)
- フロントエンド開発（React, TypeScript）
  - UI設計と実装
- インフラ運用（Google Cloud, Terraform）
  - クラウドリソースのコスト削減
- ドキュメント整備
- 保守運用
  - アプリケーション監視
  - 技術サポート

#### 課題解決

##### 課題
年末調整プロダクトにおいて、頻繁に使用される機能で3分以上かかるスロークエリが発生し、レスポンスタイムの遅延がユーザーの作業を滞らせていました。また、ピーク時にはサーバー負荷が高まり、システムの安定性にも影響を及ぼしていました。

##### 取り組み
クエリを再現し実行計画を取得することで原因を調査しました。そして、クエリの最適化やインデックスの見直しを行ないました。

##### 成果
3分以上かかるスロークエリを1秒以内にパフォーマンス改善しました。これにより、ユーザーの作業に支障なく、また、サーバー負荷が軽減され、システムの安定性も向上しました。

- 使用技術: Java8, Play Framework, JavaScript, jQuery, SQL, MySQL, nginx, AWS(EC2, Aurora MySQL, S3, CloudFront, Elastic Load Balancing, VPC, Config, Lambda), GitHub, Jenkins, Redmine, Slack

#### 主な業務と成果

- APIサーバー開発
  - DB設計、クラス設計から担当
- バッチサーバー開発
  - DB設計、クラス設計から担当
- アプリ用LPサイト開発

  - アプリケーション監視
  - インフラ管理
  - 技術サポート
- ドキュメントの整備
  - API定義書のSwagger移行
  - 機能設計書作成
  - DB設計書作成
  - インフラ構成図の最新化
  - リリース手順書作成
  - 参画時・退場時の手順書作成

#### 課題解決

##### 課題
開発リーダーとしてアサインされた案件で、仕様等に関するドキュメントがほとんどなく、システム知識が属人化していました。また、ドキュメントがないことから、改修時に考慮漏れによるバグが多く発生していました。

##### 取り組み
アサインされて以降得た知識や、テスト環境を動かしてわかった仕様をwikiにまとめることで既存機能のドキュメント作成を進めました。また、今後開発する機能についても資料として残るようにドキュメント作成の標準化を行いました。

##### 成果
チーム全体として仕様把握しやすい環境を整えられ、新規参画者への引き継ぎもスムーズに行えるようになりました。また、仕様が詳細に記載されるようになったことから考慮漏れによるバグが減り、品質向上にも繋がりました。
